# tg-parsing

Telegram-бот для мониторинга барахолок и групп: ловит объявления по ключевым словам (+ синонимы, стемминг) или фото (Groq Vision) и автоматически пишет продавцу DM.

## Два режима работы

| | 🔤 Базовый (бесплатно) | 🧠 С нейросетями (Groq) |
|---|---|---|
| Ключевые слова + синонимы | ✅ | ✅ |
| Стемминг (колонку = колонка) | ✅ | ✅ |
| Фильтр по цене | ✅ | ✅ |
| Авто-DM по шаблону | ✅ | ✅ |
| Dedup с TTL | ✅ | ✅ |
| 👁 Vision (фото → товар) | ❌ | ✅ |
| 🧬 NLP (семантика текста) | ❌ | ✅ |
| 🤖 Groq DM (генерация текста) | ❌ | ✅ |

Если `vision.api_key` пуст — AI-кнопки скрыты из интерфейса. Бот полностью работает без внешних API.

## Архитектура

| Компонент | Роль |
|-----------|------|
| **Telethon userbot** | Слушает сообщения в заданных группах от вашего аккаунта |
| **aiogram control-bot** | Управление через Telegram: меню, настройки, история |
| **Groq Vision/NLP** | Анализ фото и текста (опционально, бесплатно) |
| **SQLite** | Дедупликация DM, история совпадений, лог действий |

### Pipeline обработки сообщений

```
Сообщение в чате
  → Ключевые слова (stem + синонимы)
  → NLP семантика (если включён)
  → Vision анализ фото (если включён + есть фото)
  → Фильтр по цене
  → Dedup (25ч TTL)
  → Задержка 60–120с (human-like)
  → Авто-DM продавцу
  → Уведомление владельцу
```

## Быстрый старт

### 1. Установщик (один раз, через SSH)

```bash
git clone <repo> && cd tg-parsing
cp config.example.json config.json
# Заполнить: telegram.api_id, api_hash, phone, bot_token
# Опционально: vision.api_key (Groq) для AI-функций
pip install -r requirements.txt
python3 main.py
```

### 2. Пользователь (через Telegram)

Откройте control-бота и отправьте `/start`. Всё управление — через кнопки:

- 🔐 Авторизация userbot (QR-код или SMS-код)
- 📡 Добавить чаты и ключевые слова
- ✉️ Настроить авто-DM
- Готово — бот мониторит

## config.json — что заполняет установщик

```jsonc
{
  "telegram": {
    "api_id": 123456,          // my.telegram.org
    "api_hash": "abc...",
    "phone": "+7...",          // номер userbot-аккаунта
    "bot_token": "7...:AA..."  // токен от @BotFather
  },
  "vision": {
    "api_key": ""              // пусто = базовый режим, "gsk_..." = AI-режим
  }
}
```

Остальные поля управляются через Telegram-бота. Полный шаблон — `config.example.json`.

## Меню control-бота

### /start — Дашборд
```
📊 Активен | 2 чата | 3 слова
🔍 5 находок | ✉️ 3 DM | 💰 до 30 000₽

[📡 Мониторинг]   [✉️ Авто-DM]
[📋 История]      [⚙️ Настройки]
[⏸ Пауза]        [❓ Помощь]
```

### 📡 Мониторинг
Чаты + ключевые слова с синонимами + макс. цена — всё в одном разделе.
- **Добавить чат**: `@username`, числовой ID, или переслать сообщение из группы
- **Синонимы**: выбрать слово → добавить/удалить синонимы (через запятую)
- **Стемминг**: «колонку» автоматически матчит «колонка», «колонки» и т.д.

### ✉️ Авто-DM
- Вкл/выкл, шаблон с плейсхолдерами, задержка, повтор
- Плейсхолдеры: `{type}` `{price}` `{link}` `{author}` `{chat_title}` `{message_snippet}`

### ⚙️ Настройки (адаптивное)
- **С Groq API key**: видны кнопки Vision, NLP, Groq DM
- **Без Groq API key**: только dry-run, лимиты, списки
- 👥 **Списки**: чёрный (не писать DM) и белый (игнорировать cooldown)
- 📊 **Лимиты**: DM/час, Vision/мин, NLP/мин

## Авторизация userbot

1. `/start` → ⚙️ Настройки → 🔐 Авторизация
2. Отсканируйте QR или перейдите по ссылке на мобильном
3. Если QR не работает — бот переключится на ввод кода из Telegram

> ⚠️ **Если у вас включена 2FA**: временно отключите двухфакторную авторизацию в Telegram, авторизуйте userbot, затем включите 2FA обратно. Сессия сохраняется, повторная авторизация не нужна.

## Синонимы и keyword_map

```json
"keyword_map": {
  "колонка": ["акустика", "speaker", "JBL", "bluetooth", "саундбар"],
  "телевизор": ["телек", "экран", "плазма", "OLED"]
}
```

Синонимы управляются через бота: 📡 Мониторинг → 🏷 Синонимы.
При совпадении синонима в уведомлении показывается основное слово (ключ).

## 🔑 Groq API (опционально, бесплатно)

1. https://console.groq.com → Sign Up → API Keys → Create
2. Скопируйте `gsk_…` в `config.json` → `vision.api_key`

**Бесплатные лимиты:** 30 запросов/мин, 1 000/день, 500 000 токенов/день.

Vision вызывается **только** когда нет совпадения по тексту, но есть фото. При исчерпании лимита бот продолжает работать по ключевым словам.

## Развёртывание

### systemd (Linux/VPS)

```bash
sudo cp deploy/tg-parsing.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now tg-parsing
```

### Docker

```bash
docker compose -f deploy/docker-compose.yml up -d --build
```

### Локальные варианты

Бот работает везде — все соединения исходящие, публичный IP не нужен:

| Платформа | Примечания |
|-----------|------------|
| VPS (Ubuntu/Debian) | Рекомендуемый вариант |
| Hyper-V (Ubuntu VM) | Идентично VPS |
| Synology Docker | Container Manager или SSH + docker compose |
| Raspberry Pi | ARM, Python 3.10+ |

**Ограничения:** постоянный интернет. Если MTProto заблокирован — используйте SOCKS5 proxy (Telethon поддерживает через `PySocks`).

## Безопасность

- Сессия Telethon хранится локально в `session/`
- Коды авторизации удаляются из чата сразу после обработки
- 2FA-пароли **не принимаются** через бота — вместо этого инструкция по временному отключению
- `config.json` содержит API ключи — не коммитьте в публичный репозиторий
